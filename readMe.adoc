= Banking Fraud Management System
C++ Project – README (AsciiDoc Version with Dot Lists)

== Overview
This project implements a Banking Fraud Management System designed to detect, score, and review potentially fraudulent transactions- It uses several data structures—including a Binary Search Tree (BST), linked lists, queues, and a max-heap—to store account data, process incoming transactions, and evaluate suspicious behavior-

A graphical user interface (GUI) will allow administrators to view accounts, inspect transaction histories, and review flagged transactions-

== Core System Workflow

- Incoming transactions enter the system through a transaction queue-
- Each transaction is evaluated against multiple fraud rules:
   . Amount too large relative to the last ~5 transactions
   . Suspicious or abnormal transaction location
   . Multiple transactions in a short time window
   . Transaction from blacklisted account (automatic denial)
- Handling outcomes:
   . Safe transactions → appended to account transaction history
   . Suspicious transactions → risk score assigned → inserted into max heap
   . Blacklisted accounts → automatic denial without heap insertion
- Admin review:
   . Manager extracts highest-risk transaction from max heap
   . Approval → added to account transaction list
   . Denial → account added to blacklist
- GUI provides:
   . Full account list
   . Individual account details
   . Transaction history view
   . Indicators showing transactions under review

pointer to linked list of transactions

   . Insert an account
   . Search account by ID
   . (Optional) Delete account

   . Retrieve last N transactions
   . Efficient traversal for rule checking

== Module Interaction Overview

- Transaction Queue → Fraud Rules
   . If approved: added to BST
   . If risky: inserted into Max Heap
   . If blacklisted: automatically denied

- Max Heap → Admin Review
   . Approve: add to BST
   . Reject: blacklist account

- GUI → interfaces with all modules for display and actions
